# 数据分析 Prompt 最佳实践（已验证）

## 1) 指标诊断（先结论后证据）

**适用场景**：运营指标波动分析。  
**输入变量**：`<指标定义>` `<时间窗口>` `<数据表结构>` `<异常现象>`

```text
请分析以下指标异常并给出诊断：
- 指标定义：<指标定义>
- 时间窗口：<时间窗口>
- 数据结构：<数据表结构>
- 异常现象：<异常现象>

输出：
1. 结论摘要（不超过3条）
2. 证据链（维度拆解、对比结果）
3. 可能原因排序（高/中/低）
4. 建议的验证实验与SQL思路
```

**验证清单**

- 结论是否有证据对应
- 是否包含可执行验证步骤
- 是否避免“拍脑袋”归因

## 2) 自动化周报生成

**适用场景**：固定周期复盘汇报。  
**输入变量**：`<核心数据>` `<业务目标>` `<上周动作>`

```text
请生成周报（管理层可读）：
- 核心数据：<核心数据>
- 业务目标：<业务目标>
- 上周动作：<上周动作>

格式要求：
1. 本周结论（3条）
2. 数据看板（指标、环比、同比、解释）
3. 问题与风险
4. 下周动作（负责人、截止时间）
```

**验证清单**

- 是否面向决策者语言
- 是否包含负责人与截止时间
- 数据解释是否一致
