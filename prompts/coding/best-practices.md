# 编程 Prompt 最佳实践（已验证）

## 1) 缺陷修复（最小改动）

**适用场景**：线上 Bug 修复、回归风险高。  
**输入变量**：`<报错信息>` `<相关文件>` `<复现步骤>` `<期望行为>`

```text
你是资深工程师。请基于以下信息修复问题，并遵循“最小改动原则”：
- 报错：<报错信息>
- 涉及文件：<相关文件>
- 复现步骤：<复现步骤>
- 期望行为：<期望行为>

要求：
1. 先定位根因，再给出修复方案（不超过3点）。
2. 仅修改必要代码，保持原有接口不变。
3. 同时补充/更新测试用例覆盖该问题。
4. 输出：变更摘要、关键 diff、测试结果、潜在风险。
```

**验证清单**
- 是否说明根因而非只改症状
- 是否包含测试并可通过
- 是否避免无关重构

---

## 2) 新功能实现（先方案后代码）

**适用场景**：需求明确但实现路径多。  
**输入变量**：`<需求描述>` `<约束条件>` `<相关模块>`

```text
请实现需求：<需求描述>
上下文：<相关模块>
约束：<约束条件>

执行方式：
1. 先给出实现方案（数据结构、流程、边界处理）。
2. 再给出完整代码改动。
3. 列出至少5个测试点（正常/边界/异常）。
4. 输出兼容性影响与回滚方案。
```

**验证清单**
- 是否覆盖边界条件
- 是否说明兼容性影响
- 是否有可执行测试点
