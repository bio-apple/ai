G

ChatGPT

OpenAI 出品的通用 AI 助手，用户量最大、功能最全。支持对话、图像生成、文件分析、代码编写、自定义 GPTs 和语音交互。

快速入门

  1. 访问 chatgpt.com 注册（支持 Google/Apple 登录）
  2. 下载 iOS / Android App，开启语音对话模式
  3. 在输入框下方选择模型：Instant（快速）/ Thinking（深度推理）/ Pro（最强）
  4. 设置 → 个性化指令，告诉 ChatGPT 你的背景和偏好
  5. 上传 PDF、图片、表格让 AI 分析处理

核心功能

  • 多模型模式 — Instant 日常快速，Thinking 复杂推理，Pro 高难度任务
  • 自定义 GPTs — 创建专属 AI 助手，分享给他人使用
  • 文件与图像 — 上传文档分析、DALL·E 图像生成与编辑
  • 记忆功能 — 跨对话记住你的偏好和背景信息
  • 语音模式 — 实时语音对话，移动端体验优秀
  • Deep Research — 自动搜索多个来源，生成研究报告

Google Gemini

Google 的 AI 助手，深度整合 Gmail、Drive、YouTube 等 Google 服务。支持多模态输入、Deep Research、Gems 自定义工具和实时语音对话。

快速入门

  1. 访问 gemini.google.com，用 Google 账号登录
  2. Android 用户可下载 Gemini App 或设为默认 AI 助手
  3. 设置 → Instructions for Gemini，配置个性化指令
  4. 连接 Google 应用：Gmail、Drive、YouTube、Google 地图
  5. 在输入框切换模型：Flash（快速）/ Pro（强力）/ Ultra

核心功能

  • Gems — 创建可复用的专属 AI 工具（如写作助手、代码审查员）
  • Deep Research — 自动浏览网页，生成带引用的深度研究报告
  • Gemini Live — 实时语音对话，支持摄像头和屏幕共享
  • Canvas — 协作编辑文档、代码和图表的工作区
  • 图像与视频 — Imagen 图像生成、Veo 视频创作
  • Scheduled Actions — 定时自动执行重复性 AI 任务

DeepSeek

国产高性能 AI 模型，以极强的推理能力和极具竞争力的 API 价格著称。网页版免费使用，API 兼容 OpenAI 格式，开发者可零成本迁移。

快速入门

  1. 访问 chat.deepseek.com 免费注册使用
  2. 切换「深度思考」模式处理复杂推理和数学问题
  3. 开发者：访问 platform.deepseek.com 获取 API Key
  4. API 调用只需改 OpenAI SDK 的 base_url 为 https://api.deepseek.com
  5. 模型选择：deepseek-v4-flash（快速）/ deepseek-v4-pro（强力）

核心功能

  • 深度思考模式 — 展示推理过程，数学和逻辑题表现突出
  • 免费网页版 — chat.deepseek.com 无需付费即可使用
  • OpenAI 兼容 API — 现有 OpenAI 代码几乎零改动接入
  • 超长上下文 — V4 支持最高 1M token 上下文窗口
  • 极低 API 价格 — 约为 GPT-4 的 1/10，适合大规模部署
  • Agent 工具集成 — 支持 Claude Code、Copilot 等作为后端模型

Claude

Anthropic 出品的 AI 助手，包含网页版 Claude.ai 和终端编程工具 Claude Code。擅长复杂推理、长文档分析和高质量代码生成。

快速入门

  1. 访问 claude.ai 注册账号（支持 Google 登录）
  2. 下载桌面版 App（macOS / Windows），体验更快的响应速度
  3. 安装 Claude Code：curl -fsSL https://claude.ai/install.sh | bash
  4. 在项目目录运行 claude，按提示完成浏览器登录
  5. 创建 CLAUDE.md 文件，写入项目规范和编码约定

核心功能

  • Projects — 为每个项目创建独立工作区，上传知识库文件
  • Artifacts — 实时预览代码、文档和网页原型
  • Skills — 将重复工作流封装为可复用技能包
  • Connectors — 连接 Google Drive、Slack、Notion 等工具
  • Claude Code — 终端中自主读写文件、运行命令、提交代码
  • Extended Thinking — 深度思考模式，适合复杂问题

OpenAI Codex

Cursor

基于 VS Code 的 AI 原生代码编辑器，将大语言模型深度集成到编辑体验的每一层。支持 Tab 补全、内联编辑、Agent 自主编程和 MCP 扩展。

快速入门

  1. 访问 cursor.com/download 下载安装包
  2. 首次启动时导入 VS Code 扩展和设置
  3. 打开项目文件夹：File → Open Folder
  4. 按 Cmd+I（Mac）或 Ctrl+I（Win）打开 Agent 面板
  5. 用 @ 引用文件，给 Agent 更多上下文
  6. 在 .cursor/rules/ 中创建项目规则文件

核心功能

  • Tab 补全 — 预测式 AI 自动补全，支持多行代码
  • Cmd+K — 选中代码后内联 AI 编辑
  • Agent 模式 — 自主搜索代码库、编辑文件、运行终端
  • Plan 模式 — Shift+Tab 先制定计划再执行
  • Rules — .cursor/rules/ 定义项目编码规范
  • MCP 服务器 — 连接数据库、GitHub 等外部系统
  • Cloud Agents — 浏览器/手机远程启动编程任务

ChatGPT 入门 约 15 分钟

用 ChatGPT 写一份专业周报

不只会聊天——用结构化提示词让 ChatGPT 生成高质量的工作周报。

展开步骤
目标：输入本周工作要点，输出格式规范、语言专业的周报邮件。
  1. Step 1 — 设置个性化指令

    ChatGPT → 设置 → 个性化 → 填入你的职位、团队、汇报对象，后续对话自动带入上下文。

  2. Step 2 — 提供本周素材
    我要写本周工作周报，请帮我整理成邮件格式。本周完成： - 完成了 XX 功能开发和测试 - 参加了 XX 项目评审会议 - 修复了 3 个线上 Bug - 下周计划：开始 YY 模块开发 要求：语气专业简洁，分「本周完成」「遇到的问题」「下周计划」三节，每节用 bullet points。
  3. Step 3 — 迭代优化
    语气再正式一些，「遇到的问题」部分帮我补充一个合理的技术挑战描述，并写上解决方案。
  4. Step 4 — 保存为 GPT（可选）

    如果每周都写周报，可创建自定义 GPT「周报助手」，预设好格式和风格，以后只需粘贴要点即可。

Gemini 入门 约 20 分钟

用 Gemini Deep Research 做行业调研

让 Gemini 自动搜索多个来源，生成带引用链接的深度研究报告。

展开步骤
目标：调研一个行业话题，获得结构化的研究报告和参考来源。
  1. Step 1 — 开启 Deep Research

    访问 gemini.google.com，在工具栏选择 Deep Research（需 Gemini Advanced 订阅）。

  2. Step 2 — 提出研究问题
    请深入研究：2026年 AI 编程工具市场格局 需要涵盖：主要玩家（Cursor、Copilot、Codex、Claude Code）、市场规模、用户画像、未来趋势 输出格式：执行摘要 + 分章节正文 + 参考来源列表
  3. Step 3 — 审阅与追问

    Gemini 会花几分钟自动浏览网页并撰写报告。完成后可追问：「请补充中国市场的数据」或「对比各工具定价」。

  4. Step 4 — 导出使用

    点击导出到 Google Docs，或直接复制到 Notion / 飞书文档中分享给团队。

Cursor 入门 约 30 分钟

从零搭建待办事项 Web App

用 Cursor Agent + Plan 模式，从空文件夹到可运行的 HTML/CSS/JS 应用。

展开步骤
目标：创建一个支持添加、完成、删除任务的待办清单，数据保存在 localStorage。
准备：安装 Cursor · 新建空文件夹并打开
  1. Step 1 — 用 Plan 模式制定方案

    按 Cmd+I 打开 Agent，按 Shift+Tab 切换到 Plan 模式，输入：

    我想做一个待办事项 Web App。功能：添加任务、标记完成、删除任务、数据持久化到 localStorage。技术栈：纯 HTML + CSS + JS，不要框架。请先分析需求，列出文件结构和实现计划，不要写代码。

    💡 Plan 模式会先出方案，你确认后再让它写代码，避免返工。

  2. Step 2 — 切换到 Agent 模式实现

    确认计划后，切换到 Agent 模式，发送：

    计划没问题，请按方案创建 index.html、style.css、app.js。要求：界面简洁现代、深色主题、移动端适配。

    💡 Agent 会自动创建多个文件。在 diff 面板逐块审查，不满意的直接说「把删除按钮改成红色」。

  3. Step 3 — 本地预览和迭代

    让 Agent 启动本地服务器：

    运行 python3 -m http.server 8080 启动预览，然后检查页面是否正常工作。

    浏览器打开 http://localhost:8080，测试添加/完成/删除功能。

  4. Step 4 — 添加 Rules 固化规范

    创建 .cursor/rules/web.mdc，写入项目约定，后续 Agent 会自动遵守：

    请创建 .cursor/rules/web.mdc，内容：使用纯 HTML/CSS/JS；CSS 变量管理颜色；函数加简短注释；中文 UI 文案。
  5. Step 5 — 验收清单
    • 能添加新任务并显示在列表中
    • 点击可标记完成（划线样式）
    • 可删除任务
    • 刷新页面后数据仍在
    • 手机浏览器布局正常
Cursor 中级 约 45 分钟

给现有项目添加用户登录功能

在已有代码库上用 Agent 模式实现 JWT 认证，涵盖前后端多文件修改。

展开步骤
目标：为现有 Web 项目添加注册、登录、登出功能，用 JWT 做会话管理。
准备：一个已有的前后端项目 · Cursor 打开项目根目录
  1. Step 1 — 让 Agent 理解现有代码
    @src/ @backend/ 请分析这个项目的架构：用的什么框架、数据库、路由结构、前端如何调 API。不要改代码，只输出分析结果。
  2. Step 2 — Plan 模式设计认证方案
    基于刚才的分析，设计用户认证方案：注册/登录/登出 API、JWT 签发与验证中间件、前端登录页和路由守卫。列出要改的文件清单和实施顺序。
  3. Step 3 — 分步实现后端
    先只做后端：创建 User 模型、注册和登录 API、JWT 中间件。写完后运行现有测试，确保没有破坏原有功能。
  4. Step 4 — 实现前端页面
    实现前端登录/注册页面，登录成功后将 token 存 localStorage，API 请求自动带 Authorization header。未登录用户跳转登录页。
  5. Step 5 — 测试与修复
    运行项目的测试命令，修复所有失败用例。然后手动测试：注册新用户 → 登录 → 访问受保护页面 → 登出。

    💡 大功能一定要分步做。一次让 Agent 改太多文件容易出错，每步验证后再继续。

Claude 入门 约 20 分钟

用 Claude Projects 生成项目技术文档

上传代码仓库，让 Claude 自动分析并输出架构文档和 API 说明。

展开步骤
目标：为一个陌生代码库生成 README、架构图说明和 API 文档。
准备：Claude Pro 账号 · 项目核心源码文件（可脱敏）
  1. Step 1 — 创建 Project

    在 claude.ai 左侧点击「Projects」→「New Project」，命名如「MyApp 文档」。

  2. Step 2 — 上传知识库

    将以下文件上传到 Project 知识库：

    • package.json / requirements.txt（依赖信息）
    • 主要入口文件（如 main.py、index.ts）
    • 路由/控制器文件
    • 数据库模型文件
  3. Step 3 — 设置 Project 指令
    你是一个技术文档工程师。输出使用中文 Markdown。代码示例保留原始语言。文档结构：概述 → 架构 → 目录结构 → API 列表 → 部署说明。
  4. Step 4 — 分步生成文档

    依次发送以下提示词（不要一次全发）：

    根据知识库中的代码，生成项目概述：这个项目做什么、技术栈、核心模块关系。用 Mermaid 画架构图。
    列出所有 API 端点，格式：方法 | 路径 | 功能 | 请求参数 | 响应示例。用表格呈现。
    生成部署指南：环境要求、安装步骤、环境变量配置、启动命令。
  5. Step 5 — 导出为 Artifact

    让 Claude 将三份文档合并为一个完整 README，在 Artifacts 面板预览后复制到项目仓库。

    💡 Projects 的知识库是持久的，后续代码更新后重新上传，Claude 能基于新代码增量更新文档。

Claude 中级 约 25 分钟

用 Claude Code 终端修复 Bug

在终端用 Claude Code 定位、修复 bug 并提交 git commit，完整开发闭环。

展开步骤
目标：修复一个已知 bug：「用户列表分页到第 2 页时返回空数据」。
准备：安装 Claude Code · 克隆项目到本地 · 创建 CLAUDE.md
  1. Step 1 — 编写 CLAUDE.md

    在项目根目录创建 CLAUDE.md：

    # 项目规范 - 语言：Python 3.11 + FastAPI - 测试：pytest，运行命令 pytest tests/ - 提交格式：fix: 描述 / feat: 描述 - 不要修改数据库 migration 文件
  2. Step 2 — 启动 Claude Code
    cd your-project && claude

    首次使用会打开浏览器完成登录。

  3. Step 3 — 描述 Bug 让 AI 调查
    Bug 报告：GET /api/users?page=2 返回空数组，但数据库有 50 条记录，每页 20 条。page=1 正常。请先找到分页相关代码，分析根因，不要急着改。
  4. Step 4 — 确认方案后修复
    分析合理。请修复这个 bug，然后运行 pytest 确认测试通过。如果有分页相关测试一并更新。

    💡 Claude Code 会自动搜索代码、编辑文件、运行测试。每一步你都可以审查。

  5. Step 5 — 提交代码
    测试通过了。请 git add 相关文件并 commit，message 用 fix: pagination offset calculation for page > 1
Codex 入门 约 20 分钟

用 Codex CLI 快速搭建 REST API

从空目录开始，用 Codex 终端代理生成完整的 CRUD API 项目。

展开步骤
目标：创建一个图书管理 REST API，支持增删改查，含测试。
准备：ChatGPT 账号 · 安装 Codex CLI · Python 3.10+
  1. Step 1 — 初始化项目
    mkdir book-api && cd book-api && codex

    登录后运行 /init 生成 AGENTS.md 模板。

  2. Step 2 — 编辑 AGENTS.md
    # Book API - Python 3.11 + FastAPI + SQLite - 测试框架：pytest - 运行：uvicorn main:app --reload - 测试：pytest -v - 代码风格：type hints，函数加 docstring
  3. Step 3 — 下达任务
    创建一个图书管理 REST API。数据模型：id, title, author, isbn, created_at。端点：GET/POST /books, GET/PUT/DELETE /books/{id}。包含 pytest 测试。完成后运行测试验证。
  4. Step 4 — 审查与迭代

    Codex 会自主创建文件、安装依赖、运行测试。审查 diff 后，如需调整：

    /review — 让 Codex 自查代码质量
    给 DELETE 端点加上确认逻辑，删除前检查图书是否存在，不存在返回 404。
  5. Step 5 — 验收
    启动服务器，用 curl 测试所有端点：创建一本书 → 查询列表 → 更新 → 删除。把测试结果贴给我。
Codex 中级 约 15 分钟（异步）

用 Codex Cloud 自动处理 GitHub Issue

在 GitHub Issue 中 @codex，让云端 Agent 自动修复并提交 PR。

展开步骤
目标：将 GitHub Issue 中的 bug 报告自动转化为修复 PR。
准备：ChatGPT Pro · GitHub 仓库 · Codex Cloud 已连接 GitHub
  1. Step 1 — 配置 Codex Cloud 环境

    访问 chatgpt.com/codex → Settings → 连接 GitHub → 选择仓库 → Create environment。

    环境配置示例：

    Setup: pip install -r requirements.txt Test: pytest tests/ -v Internet: off
  2. Step 2 — 在仓库添加 AGENTS.md

    确保仓库根目录有 AGENTS.md，Codex Cloud 会自动读取：

    # 项目说明 - Python FastAPI 项目 - 所有修改必须通过 pytest - PR 标题格式：fix: 或 feat: 开头 - 不要修改 alembic/ 目录
  3. Step 3 — 创建 Issue 并 @codex

    在 GitHub 创建 Issue，描述清晰：

    ## Bug: 搜索功能大小写敏感 **复现步骤** 1. 访问 /api/search?q=hello 2. 数据库有标题 "Hello World" 的记录 3. 返回空结果 **期望**：搜索应不区分大小写 @codex 请修复此问题并提交 PR
  4. Step 4 — 等待 Codex 完成

    Codex 在云端沙箱中：拉取代码 → 分析 Issue → 修改代码 → 运行测试 → 创建 PR。通常 5–15 分钟。

    💡 你可以在 Codex Cloud 面板实时查看进度，也可以去做别的事，完成后会通知你。

  5. Step 5 — Review PR 并合并
    • 检查 PR 的 diff，确认修改范围合理
    • 查看 CI 测试是否通过
    • 本地 checkout 分支手动验证
    • 确认后 merge PR

通用实战技巧

  • 先 Plan 后 Agent — 复杂任务先让 AI 出方案，确认后再写代码
  • 小步迭代 — 每次只做一个功能点，验证通过再继续
  • 写好配置文件 — CLAUDE.md / AGENTS.md / .cursor/rules 是 AI 的「员工手册」
  • 明确验收标准 — 告诉 AI「做完后运行 xxx 测试」而不是「做好就行」
  • 审查每一行 diff — AI 会犯错，你的审查是最关键的质量关卡
  • 用 @ 引用文件 — 给 AI 精确的上下文，比笼统描述效果好 10 倍
  • 分角色使用工具 — Cursor 日常编码、Claude 分析文档、Codex 自动化 PR
  • 保存好用的提示词 — 把验证有效的 prompt 存到项目文档中复用
